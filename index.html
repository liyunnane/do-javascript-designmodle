      用function定义的函数也是全局变量。最初定义的函数办法是function aa()但此种方法也会生成全局变量aa

    • 函数的另一种形式

      var aa=function(){}是函数的另一种表现形式，但是这个方法在用的时候要提前声明。全局声明变量在团队开发中，就会容易被别人覆盖

    • 用对象收编变量

      为了减少团队中被别人覆盖的概率，可以用对象来收编变量。例如： 
                                      var checkObject={
                                    checkName:function(){
                                        //验证姓名
                                    },
                                    checkEmail:function(){
                                        //验证邮箱
                                    },
                                    checkPassword:function(){
                                        //验证密码
                                    },
                                }
      此时将所有的函数作为checkObject对象的方法，这样我们就只有一个对象，使用的时候用点语法进行访问。

    • 对象的另一种形式

      既然可以用点语法进行访问，那么也可以用点语法进行创建。在js中函数也是对象，声明一个对象然后给它添加方法 例如： 
                                      var checkObject=function(){};{}
                                checkObject.checkName=function(){
                                     //验证名字
                                 }；
                                checkObject.checkEmail=function(){
                                     //验证邮箱
                                }；
      但是这个对象类再用new创建新的对象时，新创建的对象是不能继承这些方法的。

    • 真假对象

      如果你想简单的复制一份出来，可以将这些方法放在一个函数对象中，例如： 
                                      var checkObject=function(){
                                    return{
                                        checkName:function(){
                                          //验证名字
                                        },
                                        checkEmail:function(){
                                          //验证邮箱
                                        },
                                    }
                                }
      使用的时候就是var a=checkObject();a.checkEmail()

    • 类也可以

      虽然创建的新对象完成了我们的需求，但是它不是一个真正意义上类的创建方法。并且创建的a与checkObject没有任何关系因此我们稍微改造一下例如 
                                      var checkObject=function(){
                                        this.checkName=function(){
                                          //验证名字
                                        },
                                        this.checkEmail=function(){
                                          //验证邮箱
                                        },
                                }
      使用的时候就是var a=new checkObject();a.checkEmail();如果每个人都对这个类实例化了，那么每个人都将有自己的一套方法如果每个人都对这个类实例化了

    • 一个检测类

      上述这种办法是通过this定义的，所以每一次通过new关键字创建新对象的时候，新创建的对象都会对类的this上的属性进行复制，然而有时候这么做造成的消耗是很奢侈的，因此处理成如下形式 
                                      var checkObject=function(){
                                    checkObject.prototype.checkName=function(){
                                          //验证名字
                                        },
                                    checkObject.prototype.checkEmail=function(){
                                          //验证邮箱
                                        },
                                }
      这样创建对象实例的时候，创建出来啊的对象所拥有的方法就都是一个了。因为他们都要依赖prototype原型依次寻找，而找到的方法都是同一个他们都绑定在checkObject对象类的原型上。当然写可以简化为如下形式 
                                      var checkObject=function(){
                                    checkObject.prototype={
                                        checkName:function(){
                                          //验证名字
                                        },
                                        checkEmail:function(){
                                          //验证邮箱
                                        },
                                    }
                                }
      但是这两种方式不能混着用，因为一旦混用，如果在后面为对象的原型对象复制新对象时，那么它将会覆盖掉之前对prototype对象赋值的方法。

                                      var checkObject={

                                      var checkObject=function(){

                                var a= new checkObject();
                                a..checkEmail().checkName();

    • 面向过程与面向对象

      面向过程的编程会无端在页面中添加很多局部变量，不利于团队代码维护。
      面向对象编程就是将你的需求抽象成一个对象，然后针对这个对象分析其特征（属性）和动作（方法）。这个对象我们称之为类，面向对象编程其中有一个特点就是封装。

      创建一个类

      首先声明一个函数保存在变量里，然后在这个函数的内部通过对this变量添加属性或者方法来实现对类添加属性或者方法，比如 
                                          var book=function(){
                                        this.id=id;
                                        this.bookName=id;
                                        this.price=price;
                                    }
      也可以在类的原型（类也是一个对象，所以也有原型prototype）上添加属性和方法，有两种方式：

      一种是为原型对象属性赋值例如:

                                              book.prototype.display=function(){
                                            // 展示这本书
                                        }

      另一种是将一个对象赋值给类的原型对象:

                                               book.prototype={
                                                display:function(){
                                                 // 展示这本书
                                            }
                                         }
      但是这两种方法不能混用，否则会发生后者吧前者覆盖

    • 封装

      通过this添加的属性和方法跟在prototype上添加的属性和方法有什么不同？

      回答:通过this创建的属性和方法是在当前对象上添加的，然而js是一门基于原型prototype的语言，所以每次创建一个对象时，他都有一个原型prototype用于指向其继承的属性跟方法，这样通过prototype继承的属性跟方法并不是对象自身的，所以在使用这些方法时，需要通过prototype一级一级查找来得到。这样你会发现通过this定义的属性或者方法是该对象自身所有的。所以我们每次通过类来创一个新对象时，this指向的属性和方法都会得到相应的创建。而通过prototype继承的属性或者是方法是每个对象通过prototype访问到，所以我们每次通过类创建一个新对象时，这些属性和方法不会被再次创建。

      私有属性，私有方法，共有属性，共有方法，特权方法，构造器

      通过js函数级作用域的特征来实现在函数内部创建外界就访问不到的私有化变量和私有化方法。通过new关键字实例化对象时，由于对类执行一次，所以类的内部this上定义的属性和方法自然就可以复制到新创建的对象上，成为对象公有化的属性和方法。而其中一些方法能访问到类的私有属性和方法，因而得名特权方法。而我们在通过new、关键字实例化对象时，执行了一遍类的函数，所以里面通过调用特权方法自然就可以初始化对象的一些属性了，这些在创建对象时调用的特权方法还可以看做是类的构造器。如下： 

                                  var book=function(id,name,price){
                               //私有属性
                               var num=1;
                               //私有方法
                               function checkId(){};
                               //特权方法，也称对象公有方法
                               this.getName=function(){};
                               this.getPrice=function(){};
                               this.setName=function(){};
                               this.setPrice=function(){};
                               this.copy=function(){};
                               //对象公有属性
                               this.id=id;
                               //构造器
                               this.setName(name);
                               this.setPrice(price);
                            }

      那么问题来了，在类的外部通过点语法定义的属性和方法以及在外部通过prototype定义的属性和方法又有什么作用呢？

      通过new关键字创建新对象时，由于类外面通过点语法添加的属性和方法没有执行到，所以新创建的对象中无法获取他们，但是可以通过类来使用。因此在类外面通过点语法定义的属性和方法被称为类的静态共有属性和类的静态共有方法。而类通过prototype创建的属性或者方法在类实例的对象中是可以通过this访问到的，所以我们将prototype中的属性和方法称为共有属性和共有方法。

                                      //类静态公有属性(对象不能访问)
                                book.isChinese=true;
                                //类静态公有方法(对象不能访问)
                                book.resetTime=function(){
                                    console.log('new Time')
                                };

                                book.prototype={
                                    //公有属性
                                    isJSBook:false,
                                    //公有方法
                                    display:function(){}
                                }
                                var b=new book(11,'jaaaa',50);
                                console.log(b.num);                //undefined
                                console.log(b.id);                 //11
                                console.log(b.isChinese);          //undefined
                                console.log(b.isJSBook);           //false
                                console.log(b.isJSBook1);          //false
                                console.log(book.isChinese)        //true
      有时我们经常将类的静态变量通过必报来实现
                                   var books = (function () {
                                //静态私有变量
                                var bookNum = 0;
                                //静态私有方法
                                function checkBook(name) {
                                };
                                //返回构造函数
                                return function(newId,newName,newPrice){
                                    //私有变量
                                    var name,price;
                                    //私有方法
                                    function checkId(id) {}
                                    //特权方法，也称对象公有方法
                                    this.getName = function () {};
                                    this.getPrice = function () {};
                                    this.setName = function (){} ;
                                    this.setPrice = function () {};
                                    this.copy = function () {};
                                    bookNum++;
                                    if(bookNum>100){
                                        throw new err ('我们仅出版100本书');
                                    }
                                    //对象公有属性
                                    this.id = id;
                                    //构造器
                                    this.setName(name);
                                    this.setPrice(price);
                                }
                            })();
                            books.prototype={
                                //公有属性
                                isJSBook:false,
                                //公有方法
                                display:function(){}
                            }

      闭包是有权访问另一个函数作用域中变量的函数，即在一个函数内部创建另一个函数，我们将这个闭包作为创建对象的构造函数。这样它既是闭包，有事可实例对象的函数既可以访问到类函数作用域中的变量，比如bookName,此时这个变量叫做叫做静态私有变量。但是在闭包外部添加原型属性和方法看上去像是脱离了闭包这个类，所以在闭包内部实现一个完整的类然后将其返回。如下：

                                   var books = (function () {
                                //静态私有变量
                                var bookNum = 0;
                                //静态私有方法
                                function checkBook(name) {
                                };
                                //返回构造函数
                                function _book(newId,newName,newPrice){
                                    //私有变量
                                    var name,price;
                                    //私有方法
                                    function checkId(id) {}
                                    //特权方法，也称对象公有方法
                                    this.getName = function () {};
                                    this.getPrice = function () {};
                                    this.setName = function (){} ;
                                    this.setPrice = function () {};
                                    this.copy = function () {};
                                    bookNum++;
                                    if(bookNum>100){
                                        throw new err ('我们仅出版100本书');
                                    }
                                    //对象公有属性
                                    this.id = id;
                                    //构造器
                                    this.setName(name);
                                    this.setPrice(price);
                                }
                                books.prototype={
                                   //公有属性
                                   isJSBook:false,
                                   //公有方法
                                   display:function(){}
                                };
                                //返回类
                                return _book;
                            })();

      创建对象的安全模式

      如果用构造函数实例化对象，那么经常容易忘记使用new关键字，而new关键字的作用可以看做是对当前对象的this不停的赋值，比如:

                                  var book=function(title,time,type){
                                this.title=title;
                                this.time=time;
                                this.type=type;
                            };
                            var book1=book('js','2014','jssss');

      假设直接这样子实际上是在调用函数，而这个函数是在全局作用域中执行的。那么当执行this.title时，this指向的就是全局变量window，因此console.log(book1)的结果是undefined而console.log(window.title);的结果是js那么如何避免使用new关键字来实例化对象呢？如下:

                                  var book=function(title,time,type){
                                //判断执行过程中this是否是当前这个对象（如果是则说明是用new创建的）
                                if(this instanceof book){
                                    this.title=title;
                                    this.time=time;
                                    this.type=type;
                                }else{
                                    return new book(title,time,type);
                                }
                            };
                            var book1=book('js','2014','jssss');
                            console.log(book1.title);
                            console.log(window.title);

    • 继承

        每个类都有三个部分，第一部分是构造函数内部的这是提供实例化对象复制用的。第二部分是构造函数外的直接通过点语法添加的，着是供类使用的。实例化对象是范文不到的，第三部分是类的原型中的，实例化对象可以通过其原型链间接的访问到。也是供所有实例化对象所共用的。

        六种继承方式

      •    类式继承如下

           

        类的原型对象的作用就是为类的原型添加共有方法，但是类不能直接访问这些属性和方法，必须通过原型prototype来访问。而我们实例化一个父类的时候，新创建的对象复制了父类的构造函数内的属性和方法并且将原型_proto_指向了父类的原型对象。这样就拥有了父类的原型对象上的属性和方法，并且这个新创建的对象可直接访问到父类原型对象上的属性和方法。如果我们将这个新创建的对象赋值给子类的原型，那么子类的原型就可以访问到父类的原型属性和方法。

        类式继承原理：新创建的方法不仅可以访问父类原型上的属性和方法
